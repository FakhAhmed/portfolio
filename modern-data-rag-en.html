Modern Data Stack & RAG

About the project:

This project implements a complete data processing architecture (ELT) coupled with a generative AI system (Advanced RAG). It integrates a real-time mathematical audit to evaluate and block AI hallucinations, applied to the analysis of IT support tickets.


Why this project? This PoC demonstrates the ability to build a robust bridge between traditional data engineering (Modern Data Stack) and cutting-edge generative AI. It proves to enterprises that a RAG system can be deployed in production with confidence: the data is clean (thanks to dbt), the infrastructure is scalable (GCP Serverless), and above all, the AI's responses are mathematically measured and validated to prevent any hallucinations.


Key Features:
  • Advanced RAG (Retrieval-Augmented Gen): Use of LlamaIndex for high-precision semantic search on an IT support ticket database.
  • Mathematical Audit (MLOps): Integration of real-time evaluators (RAGAS type) to visually calculate the Faithfulness (zero hallucinations) and Relevancy score of each AI-generated response.
  • Data Engineering (ELT): Raw data transformation pipeline using dbt Core directly within Google BigQuery (cleaning, date standardization, integrity testing).
  • Scalable Vectorization: Use of Google's embedding model (Vertex AI) and storage in Qdrant Cloud for ultra-fast vector search.
  • 100% Serverless Automation: Replacement of heavy orchestrators (like Airflow) with a native GCP architecture combining Cloud Scheduler and Cloud Run Job for automated and cost-effective daily execution.
  • User Experience (UI/UX): Smooth Streamlit interface including an AI audit dashboard, direct display of sources (reference tickets), and secure asynchronous management.

Project Structure:
  • /app/ : "The Frontend". Contains the Streamlit user interface (main.py) and the display of the mathematical evaluation dashboard.
  • /dbt_transform/ : "The Data Factory". Complete dbt project containing SQL cleaning models and BigQuery configuration.
  • /scripts/ : "The Laboratory". Initial engineering scripts for mock data generation and local AI evaluation tests.
  • run_pipeline.py : Main entry point of the data pipeline (triggers dbt then ingestion into Qdrant).
  • Dockerfile.app & Dockerfile.pipeline : Separate containerization files adhering to Monorepo best practices.
