Streaming Pipeline (Kafka)

This project is currently in progress.

About the project:

A pure Data Engineering project aiming to demonstrate the ability to process high-velocity data streams in real-time (millisecond latency). The architecture will replace traditional batch processing approaches to focus on the ingestion, transformation, and storage of continuous asynchronous data.


Planned Objectives and Features:
  • Event Ingestion: Implementation of asynchronous producers (Event Brokers) managing high data throughput.
  • On-the-fly Processing: Transformation, temporal aggregation, and enrichment of data in transit.
  • Lakehouse Storage: Robust saving respecting ACID properties for the historization of streaming-read data.
  • Fault Tolerance: Offset management and incident recovery inherent to the distributed architecture.

Planned Tech Stack:
  • Apache Kafka
  • Apache Flink
  • Delta Lake / Iceberg
  • Docker Compose
Architecture diagrams and source code will be published soon.
