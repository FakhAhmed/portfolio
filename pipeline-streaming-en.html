Streaming Pipeline & Embedded ML

About the project:

A comprehensive high-velocity streaming data processing architecture coupled with a real-time artificial intelligence system (In-Line ML Inference). This project demonstrates the implementation of an end-to-end Big Data pipeline for ingestion, complex temporal analysis, and instant bank fraud detection, orchestrated on the Apache and Google Cloud ecosystems.


Why this project? This PoC proves that it is possible to stop a fraudulent transaction in milliseconds (thanks to Kafka and Flink) while ensuring ACID transactional properties on the Data Lake (thanks to Iceberg) for future analytics. It illustrates the perfect fusion of streaming Data Engineering and the industrialization of Machine Learning models (MLOps) in a constrained environment.


Key Features:
  • Interface Contract (Governance): Use of Apache Avro and Confluent Schema Registry to validate and reject at the source any data not respecting the established schema, thus protecting the Data Lake.
  • Stateful Processing & Time Windows: Use of Apache Flink to track fraudsters' behavior over time by computing complex aggregations on sliding windows (Hop Windows) and managing network delays (Watermarks).
  • Embedded ML Inference (In-Line ML): Injection of an Artificial Intelligence model (Python UDF) directly into the Flink executor's memory (TaskManager) to score fraud probability in under a millisecond without external network calls.
  • Resilience & Multi-Sink Architecture: Automatic isolation of corrupted events to a Dead Letter Queue (DLQ) and asynchronous routing to simultaneously dispatch cold data (GCP/Iceberg) and trigger real-time alerts via Kafka.

Project Structure:
  • /producer/ : The "Generator". Contains the bank simulation script, the strict Avro data schema definition, and the listening microservice for real-time SMS simulation.
  • /flink_jobs/ : The "Streaming Engine". Contains the core architecture (PyFlink SQL), the Machine Learning function (Python UDF), and JAR connectors to Kafka and Google Cloud.
  • docker-compose.yml : Local infrastructure deploying Flink computation nodes, the Kafka cluster, and the Schema Registry.

Technical Architecture:
