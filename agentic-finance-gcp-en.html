Financial Agentic AI

About the project:

An enterprise-grade AI agent architecture capable of analyzing terabytes of financial data, reading complex reports, and querying markets in real-time—all fully secured, traced, and deployed on Google Cloud Platform.


Why this project? This PoC demonstrates the ability to transform raw data (unstructured, massive, and real-time) into actionable financial insights using an autonomous agent architecture. It integrates industry best practices: security by design (Guardrails), total observability (MLOps), and Cloud scalability.


Key Features:
  • Intelligent Routing (Multi-Agent): An AI supervisor dynamically delegates requests among 4 specialized experts:
    • 📊 SQL Agent: Big Data analysis with self-correcting queries.
    • 📄 RAG Agent: Vector search within complex financial documents.
    • 📈 Markets Agent: Real-time API connection to stock market prices.
    • 💬 Conversational Agent: General interactions and summaries.
  • "Enterprise-Grade" Security (Guardrails): An AI firewall (Bouncer) analyzes every incoming request to block Prompt Injection attacks and off-topic queries even before graph execution.
  • Real-World Connection: Integration of the Yahoo Finance API (yfinance) to fetch stock prices up to the second.
  • Observability & MLOps: Full integration of LangSmith for live tracing, accurate cost-per-token calculation, and performance monitoring (agent latency).
  • Big Data Engineering: Real-time querying of massive databases (e.g., Bitcoin blockchain) via BigQuery and Google Standard SQL.
  • Persistent Memory: Use of LangGraph Checkpointer to maintain conversation context from start to finish.
  • User Experience (UI/UX): Smooth Streamlit interface with asynchronous streaming to display the AI's step-by-step reasoning.

Project Structure:
  • /agent/ : The "Brain" of the application (LangGraph). Contains the routing logic (main.py) and AI tool configuration (rag_tool.py, tools.py).
  • /app/ : Asynchronous web user interface (app.py via Streamlit).
  • /data/ : Local storage of source data (PDF reports, TXT/CSV files) and the ChromaDB vector database.
  • /infra/ : Infrastructure as Code (Terraform) for automated deployment of resources on Google Cloud (main.tf).
  • Dockerfile & requirements.txt : Application containerization recipe for Cloud Run and the list of Python dependencies.
  • setup_bq.py & setup_db.py : Data engineering scripts to initialize the SQL and BigQuery databases.

Technical Stack:
  • Gemini Flash (Vertex AI)
  • LangChain & LangGraph
  • BigQuery & ChromaDB
  • LangSmith (MLOps)
  • Google Cloud Run
  • Docker
  • Terraform
  • Streamlit
