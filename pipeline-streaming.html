Pipeline Streaming (Kafka)

Ce projet est actuellement en cours de réalisation.

À Propos du projet :

Un projet pur de Data Engineering visant à démontrer la capacité de traiter des flux de données à haute vélocité en temps réel (millisecondes). L'architecture remplacera les approches classiques de traitement par lots (Batch) pour se concentrer sur l'ingestion, la transformation et le stockage de données asynchrones continues.


Objectifs et Fonctionnalités Prévues :
  • Ingestion d'événements : Mise en place de producteurs asynchrones (Event Brokers) gérant de hauts débits de données.
  • Traitement à la volée : Transformation, agrégation temporelle et enrichissement des données en transit.
  • Stockage Lakehouse : Sauvegarde robuste respectant les propriétés ACID pour l'historisation des données lues en streaming.
  • Tolérance aux pannes : Gestion des offsets et reprise après incident inhérente à l'architecture distribuée.

Stack Technique Envisagée :
  • Apache Kafka
  • Apache Flink
  • Delta Lake / Iceberg
  • Docker Compose
Les schémas d'architecture et le code source seront publiés prochainement.
