Usine de Fine-Tuning (SLM)

À Propos du projet :

Une architecture Machine Learning de bout en bout, du versioning des données jusqu'au fine-tuning d'un Small Language Model (SLM) et son déploiement web. Ce projet démontre la mise en place d'un pipeline MLOps complet pour spécialiser un modèle de langage puissant (Llama-3 8B) sur des tâches d'analyse de sentiment financier, orchestré sur l'écosystème Google Cloud et Hugging Face.


Pourquoi ce projet ? Ce PoC illustre la complexité réelle de l'ingénierie autour des Large Language Models (LLMOps). Il prouve la capacité à dépasser le simple script Python pour construire une véritable "usine" logicielle : capable de gérer des données versionnées, d'optimiser l'entraînement de milliards de paramètres, et de traquer rigoureusement les expérimentations sur le Cloud.


Fonctionnalités Clés :
  • Fine-Tuning PEFT / LoRA : Spécialisation du modèle fondation Meta Llama-3 sans ré-entraîner tous les poids. Utilisation de la quantification 4-bit (Unsloth) pour une optimisation drastique de la VRAM (entraînement sur GPU T4).
  • Versioning des Données (DVC) : La donnée financière est traitée comme du code. Son historique est strictement versionné via Data Version Control (DVC) avec un stockage distant sécurisé sur Google Cloud Storage.
  • Tracking d'Expérimentations (MLflow) : Déploiement d'un serveur MLflow propriétaire sur Google Cloud Run (Serverless). Chaque métrique (Loss) est enregistrée en direct dans le cloud.
  • Inférence Interactive (Hugging Face) : Le modèle fine-tuné est hébergé publiquement et interrogé via une interface web fluide développée avec Gradio, prouvant la capacité de "Serving" du modèle.

Structure du Projet :
  • /data/ : Espace de stockage des jeux de données d'entraînement, trackés et versionnés par DVC.
  • /notebooks/ : Laboratoire d'entraînement central exécutant le fine-tuning Unsloth et poussant les métriques vers le serveur MLflow distant.
  • /src/ : Code source de l'interface web (Gradio). Charge dynamiquement le modèle de base et l'adaptateur LoRA.
  • dvc.yaml : Le cerveau du pipeline de données vers GCP.

Architecture Technique :
  • Llama-3 8B / PyTorch
  • LoRA / Unsloth
  • DVC (Versioning Data)
  • MLflow (Tracking)
  • GCP Cloud Run
  • GCP Cloud Storage
  • Hugging Face Hub
  • Gradio (UI Web)
  • Python / TRL
